## 2. 阶段 B（36 试次，距离判断）

### 2.1 任务

显示一对石块 `X ↔ Y`（PNG 贴图 + 中文标签），问"这两块石头之间最少需要走几步？"

```
①  1 步（相邻，一步直达）
②  2 步
③  3 步
④  4 步（最远）
```

> **"一步" 的定义**：在 3×3 网格上按一次 **公交 Q / E** 或 **地铁 A / D** 方向键，刚好走到相邻一格。
> 本阶段 **不考虑快速巴士 W**——W 是同一图上的另一类捷径（仅 4 角顺时针），
> 但 Phase B 的距离按纯网格图（公交 Q/E + 地铁 A/D）计算，便于检验被试对欧氏 / 曼哈顿结构的内部表征。

### 2.2 36 试次按真值距离分类（3×3 Manhattan）

| 距离 | 数量 | 配对（共 36） |
| --- | --- | --- |
| **d = 1** | 12 | 公交 6 对 + 地铁 6 对 |
| **d = 2** | 14 | 同行隔 1（如 1-3）、同列隔 1（如 1-7）、对角邻（如 1-5、2-4、…） |
| **d = 3** | 8 | 跨两步路径（如 1-6, 1-8, 2-9, 3-4, 4-9, 6-7, 2-7, 3-8） |
| **d = 4** | 2 | 对角顶点：1-9, 3-7 |

### 2.3 平衡

- 全枚举 C(9,2) = 36 → 每节点恰好出现 8 次。
- 答案分布：①12 ②14 ③8 ④2。
- 试次顺序：被试间随机；连续相同真值 ≤ 3。
- 节点呈现位置（左/右）：每试随机翻转。

## 3. 阶段 D — 动作序列流畅度（20 试次）

### 3.1 显示

```
   [stone_01.png]   ──公交E→   [stone_02.png]   ──地铁D→   [stone_05.png]   ──公交Q→   [stone_04.png]
```

每段箭头按真实方向键的颜色绘制（深蓝/浅蓝 = 公交 Q/E；深绿/浅绿 = 地铁 A/D；紫 = 快速巴士 W）。

5 级 Likert："这一连串动作像不像一个习惯成自然的整体？"

### 3.3 关键指标（与 navigation 版定义一致）

- **GridScore** = `mean(G+F)`
- **RingScore** = `mean(L)`
- **GridRingBias** = `mean(F) − mean(L)`
- **MixedScore** = `mean(M)`

## 5. UI 风格（crafting 语境）

- **节点**：石块 PNG 贴图（`shared/assets/stone/stone_0X.png`，由 Flask `/stone/<filename>` 路由提供）+ 中文「石块一/…/九」标签；无颜色 / 形状语义，与 NC_MIX_TASK crafting 一致。
- **边**：与 navigation 完全相同的颜色和方向键编码（公交 Q/E、地铁 A/D、快速巴士 W）。
- **行动序列**（阶段 D）显示为 `[stone png] ──彩色箭头(公交Q/E/地铁A/D/快速巴士W)→ [stone png]`。
- **图例**：屏幕底部固定显示
  `——深蓝—— 公交 Q  /  ——浅蓝—— 公交 E  /  ——深绿—— 地铁 A  /  ——浅绿—— 地铁 D  /  ——紫—— 快速巴士 W`。
