## 2. 阶段 B（36 试次，距离判断）

### 2.1 任务

显示一对车站 `X ↔ Y`，问 "这两个车站之间最少需要走几步？"

```
①  1 步（相邻，一步直达）
②  2 步
③  3 步
④  4 步（最远）
```

> **"一步" 的定义**：在 3×3 网格上按一次 **公交 Q / E** 或 **地铁 A / D** 方向键，刚好走到相邻一格。
> 本阶段 **不考虑快速巴士 W**——W 是同一图上的另一类捷径（仅 4 角顺时针），但 Phase B 的距离按纯网格图（公交 Q/E + 地铁 A/D）计算，便于检验被试对欧氏 / 曼哈顿结构的内部表征。

### 2.2 36 试次按真值距离分类（3×3 Manhattan）

| 距离 | 数量 | 配对（共 36） |
| --- | --- | --- |
| **d = 1** | 12 | 公交 6 对 + 地铁 6 对 |
| **d = 2** | 14 | 同行隔 1（如 1-3）、同列隔 1（如 1-7）、对角邻（如 1-5、2-4、…） |
| **d = 3** | 8 | 跨两步路径（如 1-6, 1-8, 2-9, 3-4, 4-9, 6-7, 2-7, 3-8） |
| **d = 4** | 2 | 对角顶点：1-9, 3-7 |

### 2.3 平衡

- 全枚举 C(9,2) = 36 → 每节点恰好出现 8 次。
- 答案分布：①12 ②14 ③8 ④2（按 3×3 网格直径 4 的真实分布）。
- 试次顺序：被试间随机；连续相同真值 ≤ 3。
- 节点呈现位置（左/右）：每试随机翻转。

## 3. 阶段 D — 动作序列流畅度（20 试次）

### 3.1 显示

```
   ▲红   ──公交E→   ■蓝   ──地铁D→   ★紫   ──公交Q→   ◆橙
```

每段箭头按真实方向键的颜色绘制（深蓝/浅蓝 = 公交 Q/E；深绿/浅绿 = 地铁 A/D；紫 = 快速巴士 W）。

5 级 Likert："这一连串动作像不像一个习惯成自然的整体？"
①完全是临时拼出来的 → ⑤反射式整体（脱口而出）。

### 3.2 5 类共 20 试次

> 序列层面与旧版完全一致；下表「模式」一栏改用类型粒度（公交 / 地铁 / 快速巴士）描述，便于快速识别 chunk 类型。

#### G — grid corner chunk（5 试，3 节点 1 拐）

| # | 序列 | 模式 |
|---|---|---|
| D-G1 | 1→2→5 | 公交→地铁 |
| D-G2 | 5→6→3 | 公交→地铁 |
| D-G3 | 7→4→5 | 地铁→公交 |
| D-G4 | 9→6→5 | 地铁→公交 |
| D-G5 | 4→5→8 | 公交→地铁 |

#### F — face loop（5 试，4 节点沿单格 4-cycle）

4 个 grid 4-cycles：{1,2,5,4}, {2,3,6,5}, {4,5,8,7}, {5,6,9,8}

| # | 序列 | 模式 |
|---|---|---|
| D-F1 | 1→2→5→4 | 公交→地铁→公交 |
| D-F2 | 2→3→6→5 | 公交→地铁→公交 |
| D-F3 | 4→5→8→7 | 公交→地铁→公交 |
| D-F4 | 5→6→9→8 | 公交→地铁→公交 |
| D-F5 | 4→1→2→5 | 地铁→公交→地铁 |

#### L — ring loop（4 试，沿快速巴士 W 方向）

| # | 序列 | 模式 |
|---|---|---|
| D-L1 | 1→3→9 | 快速巴士W→快速巴士W |
| D-L2 | 3→9→7 | 快速巴士W→快速巴士W |
| D-L3 | 9→7→1 | 快速巴士W→快速巴士W |
| D-L4 | 1→3→9→7 | 快速巴士W→快速巴士W→快速巴士W |

#### S — straight（3 试，3 节点同线路）

| # | 序列 | 模式 |
|---|---|---|
| D-S1 | 1→2→3 | 公交→公交 |
| D-S2 | 1→4→7 | 地铁→地铁 |
| D-S3 | 4→5→6 | 公交→公交 |

#### M — mixed（3 试，含快速巴士 W 的混合宏）

| # | 序列 | 模式 |
|---|---|---|
| D-M1 | 1→3→6→5 | 快速巴士W→地铁→公交 |
| D-M2 | 9→7→8→5 | 快速巴士W→公交→地铁 |
| D-M3 | 7→1→2→5 | 快速巴士W→公交→地铁 |

### 3.3 关键指标

- **GridScore** = `mean(G+F)`
- **RingScore** = `mean(L)`（即沿快速巴士 W 的 chunk 化）
- **GridRingBias** = `mean(F) − mean(L)`
- **MixedScore** = `mean(M)`：跨线路 chunk 化能力

## 5. UI 风格（与 NC_MIX_TASK 对齐）

- **节点**：彩色几何形状（unicode 字符 + CSS 颜色），与 `navigation/app/common/station_names.py::STATION_SHAPES` 一一对应。不再用水果 emoji。

  `——深蓝—— 公交 Q  /  ——浅蓝—— 公交 E  /  ——深绿—— 地铁 A  /  ——浅绿—— 地铁 D  /  ——紫—— 快速巴士 W`。
